
Fraud Detection Analysis (SQL)

 Project Overview
This project focuses on analyzing transaction data to identify fraud patterns using SQL.

 Dataset Information
- Total transactions: 10,000
- Fraud transactions: 151
- Fraud rate: ~1.5%

 Key Analysis Performed
- Total transactions analysis
- Fraud vs non-fraud comparison
- Average transaction amount analysis
- High-risk merchant category identification
- Hour-wise fraud pattern analysis

 Key Insights
- Fraud transactions have a higher average amount compared to non-fraud transactions
- Fraud rate is low, but financial impact is significant
- Certain merchant categories show higher fraud risk

 Tools Used
- MySQL
- SQL (Aggregations, Group By, Filtering)
